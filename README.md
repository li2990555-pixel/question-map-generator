# 问题导图生成器

边读社科人文书，边生成问题链，输出 **Obsidian Canvas 可视导图** + **HTML 静态图** 两种格式。

## 是什么

任何一本社科人文书，本质都是在回答一个或多个核心问题。这个 Skill 帮你边读边问，把知识点转化成「问题→回答→局限→下一问题」的因果链条，并生成可视化的问题导图。

## 适用类型

| 类型 | 核心问题模板 |
|------|------------|
| 文学史 | 历史条件 → 时代问题 → 思潮回答 → 局限 |
| 文学理论 | 既有理论困境 → 核心争辩 → 理论方案 → 盲点 |
| 哲学 | 既有哲学困境 → 前人哪错了 → 新框架 → 边界 |
| 历史 | 时代背景 → 核心矛盾 → 历史进程 → 启示 |
| 社会学 | 社会现象 → 背后结构 → 理论解释 → 解释边界 |

## 使用方法

在 WorkBuddy 中调用：

```
@skill:"问题导图生成器" @"/path/to/book.pdf" 我想开始阅读这本书
```

## 输出结构

每次更新都会同时生成两种格式的文件：

```
docs/问题导图/
  ├── {书名}/
  │   ├── {书名}.canvas          ← Obsidian Canvas 文件
  │   └── cards/
  │       ├── Q1.md              ← 问题卡片
  │       ├── A1.md              ← 回答卡片
  │       ├── L1.md              ← 局限卡片
  │       └── ...
  └── {书名}_问题导图.html       ← 浏览器直接打开的静态图
```

### Canvas 文件（Obsidian）

- 横向流 + 纵向堆叠，每轮 Q→A→L 从左到右展开
- 用 Group 节点包裹每轮，颜色交替区分
- 卡片用独立 `.md` 文件引用，Obsidian 完整渲染，不截断文字
- 需 Obsidian 打开使用

### HTML 静态图（浏览器）

- 白底卡片 + 彩色左边框（Q 棕红 / A 绿 / L 橙棕）
- 每轮横向 Q→A→L，轮间细线 +「引出」标签连接
- 含 `ROUND X · 章节` 标题，完整文字可直接阅读
- **无需任何软件，双击即用浏览器打开**
- 样式内联，离线也能正常显示

两种格式的文字内容均来自同一套 `cards/` 目录下的 `.md` 文件，修改卡片后重新生成即可同步。

### 星丛式关系图（读后总结）

读完全书后，**基于全部问题链重组全书内容**，将散落各章的论断拼成一张要素间的因果网络图。

**功能：**
- 提取各轮 A 卡片的核心概念，找出要素间的清晰关系、模糊关系和缺失关系
- 不强迫一条线性因果链，承认"星座"（constellation）——所有要素同时存在、相互催化
- 输出 HTML 静态图，与问题导图保持同一视觉风格
- 附两张对照表格：「清晰关系」+「模糊/缺失关系」

**输出路径：** `docs/问题导图/{书名}_星丛式关系图.html`

**本质：** 问题导图不是"复述书的内容"，而是"用书的内容回答我的问题"。星丛式关系图则是"基于我的问题链条，重新组织全书要素间的相互关系"。

### 读书会模式（多人会议）

适合 2-4 人线上/线下读书会。支持完整的**会前→会中→会后**闭环。

**会前——多源导入问题：**
- 每个成员阅读各自负责的章节，随时记下自己的问题
- 将所有人的问题合并导入 Skill，自动标注来源（队友1/队友2/你）
- 对每个问题走标准的 Q→A→L 流程，生成「问题链 A」
- 输出会议版 HTML，按来源分组排列，卡片左上角标注来源标签

**会中——共享讨论：**
- 共享屏幕，三人同时看同一份按来源分组的「问题地图」
- 主持人引导推进，不按章节、按问题走
- 卡片不显示回答和局限（给讨论留白），只显示问题+来源

**会后——对比分析：**
- 导入会上讨论产生的新问题/新洞察（「问题链 B」）
- 生成对比分析 HTML，三列并排展示：
  - **延续**：同一问题从 A→B 有深化（答案被补充/修正/推翻）
  - **新增**：B 中有但 A 中没有的问题（会上产生的新追问）
  - **搁置**：A 中有但 B 中没有提到的问题（会上没讨论到）
- 底部总结：有效性判断 + 偏差建议

**新增输出路径：**
```
docs/问题导图/
  ├── {书名}_问题导图_会议版.html       ← 按来源分组的讨论用 HTML
  └── {书名}_问题链对比分析.html        ← 会前 vs 会后对比
```
