# 问题导图生成器

社科人文书读不完、记不住？不如把它变成一张**问题链导图**。

这个 Claude Code Skill 帮你边读边做——每读完一节，提炼一组「**问题→回答→局限→下一问题**」卡片，自动生成 **Obsidian Canvas 可视导图** + **HTML 静态图** 两种格式。一本书读完，知识脉络一目了然。

## 效果一览

在 Obsidian Canvas 中呈现的效果：

```
┌─────────────────────────────────────────────────────────────┐
│ 📖 第1轮问题链                                                │
│                                                              │
│  ❓ Q1: 启蒙理性为何无法     💡 启蒙理性的局限     ⚠️ 局限   │
│  满足人类情感需求？ ──回应──→  核心回答：理性主  ──局限──→  指出缺陷未给    │
│                              义忽视了情感…                    替代方案…       │
│                                                              │
│                                                   引出       │
└───────────────────────────────────────────────────┼──────────┘
                                                    ↓
┌─────────────────────────────────────────────────────────────┐
│ 📖 第2轮问题链                                                │
│                                                              │
│  ❓ Q2: 赫尔德如何挑战      💡 赫尔德：归属与表达  ⚠️ 局限   │
│  启蒙普世主义？ ──回应──→   核心回答：文化多元  ──局限──→  相对主义走向    │
│                              不可通约…                       孤立？          │
└─────────────────────────────────────────────────────────────┘
```

每轮包含三张卡片（紫色=问题、绿色=回答、橙色=局限），用带标签的箭头串联，轮次之间用底色 Group 区分。

## 核心理念

任何一本社科人文书，本质上都在回答一个或多个核心问题。阅读不是被动吸收，而是主动追问：

| 步骤 | 卡片 | 你在做什么 |
|------|------|-----------|
| **读前** | ❓ **问题** | 问自己：这章/这本书想解决什么问题？ |
| **读中** | 💡 **回答** | 提炼：作者给出的核心回答是什么？ |
| **读后** | ⚠️ **局限** | 追问：这个回答遗漏了什么？ |
| **继续** | ❓ **下一问题** | 把局限变成下一个起点问题 |

这个循环正是学术思考的核心——**找到一个回答，然后找到它的边界**。

## 适用书型

| 类型 | 问题模板 |
|------|---------|
| 文学史 | 历史条件 → 时代问题 → 思潮回答 → 局限 |
| 文学理论 | 既有理论困境 → 核心争辩 → 理论方案 → 盲点 |
| 哲学 | 既有哲学困境 → 前人哪错了 → 新框架 → 边界 |
| 历史 | 时代背景 → 核心矛盾 → 历史进程 → 启示 |
| 社会学 | 社会现象 → 背后结构 → 理论解释 → 解释边界 |
| 通用 | 阅读背景 → 作者想解决什么 → 他的论证 → 没回答的 |

## 如何使用

### 在 Claude Code 中

安装本 Skill 后，直接对话：

> `/问题导图 《浪漫主义的根源》`
>
> `/问题导图 浪漫主义 第三章 康德的贡献`
>
> `/问题导图`（不指定，从头开始）

AI 会像读书伴侣一样一步步引导你：先确认起点问题，你读一段后它帮你生成卡片，再引导你找局限，如此循环。


## 输出结构

每次更新同时生成两种格式的文件：

```
docs/问题导图/
  ├── {书名}/
  │   ├── {书名}.canvas          ← Obsidian Canvas 文件
  │   └── cards/
  │       ├── Q1.md              ← 问题卡片
  │       ├── A1.md              ← 回答卡片
  │       ├── L1.md              ← 局限卡片
  │       ├── Q2.md              ← 下一轮
  │       └── ...
  └── {书名}_问题导图.html       ← 浏览器直接打开的静态图
```

### Canvas 文件（Obsidian）

- 水平流 + 纵向堆叠，每轮 Q→A→L 从左到右展开
- Group 节点包裹每轮，颜色交替区分
- 卡片用独立 `.md` 文件引用，Obsidian 完整渲染，不截断文字
- 需 Obsidian 打开查看

### HTML 静态图（浏览器）

- 白底卡片 + 彩色左边框（Q 紫 / A 绿 / L 橙）
- 每轮横向 Q→A→L，轮间细线 +「引出」标签连接
- 含 `ROUND X · 章节` 标题，完整文字直接可读
- **无需任何软件，双击即用浏览器打开**
- 样式内联，离线也能正常显示

两种格式的文字内容均来自同一套 `cards/` 目录下的 `.md` 文件，修改卡片后重新生成即可同步。

## 使用流程速览

1. **AI 识别书型** → 匹配问题模板
2. **定起点问题** → 你选方向，AI 帮忙措辞
3. **读一段，填一张** → 问题 → 回答 → 局限，一次只填一张
4. **局限变起点** → 自动续接下一轮
5. **Canvas + HTML 同步更新** → 每次追加新节点，已有内容不动

## 设计原则

- **你填空，AI 不代笔** — 核心回答和局限由你自己写，AI 只给骨架和提示
- **一次一步** — 读一段生成一个节点，不一口气生成整章
- **不覆盖** — 已有文件只追加，不修改历史
- **跳过友好** — 不想填局限？先空着，以后回来补
