# 问题导图生成器

任何一本社科人文书，本质都是在回答一个或多个核心问题。这个 Skill 帮你边读边问自己四个问题，把知识点转化成"问题→回答→局限→下一问题"的因果链条，并生成可视化的 Obsidian Canvas 文件。

## 触发方式

用户主动调用，如：
- `/问题导图 浪漫主义`
- `/问题导图 《纯粹理性批判》`
- `/问题导图`（不指定章节，从头开始）

## 准备工作

1. 创建输出目录：`docs/问题导图/`
2. 如果用户指定了书名/章节，读取已有的 `.canvas` 文件（如果有），确定当前进度

## 工作流程

### Step 0: 识别书本类型

根据用户输入判断书的大类：

| 类型 | 识别信号 | 核心问题模板 |
|------|---------|-------------|
| 文学史 | "文学史"、"浪漫主义"、"现实主义" | 历史条件 → 时代问题 → 思潮回答 → 局限 |
| 文学理论 | "文学理论"、"文论"、"本质论" | 既有理论困境 → 核心争辩 → 理论方案 → 盲点 |
| 哲学 | "哲学"、"形而上学"、"认识论" | 既有哲学困境 → 前人哪错了 → 新框架 → 边界 |
| 历史 | "历史"、"通史"、"近代史" | 时代背景 → 核心矛盾 → 历史进程 → 启示 |
| 社会学 | "社会学"、"社会"、"结构" | 社会现象 → 背后结构 → 理论解释 → 解释边界 |
| 通用（兜底） | 其他 | 阅读背景 → 作者想解决什么问题 → 他的论证 → 没回答的 |

如果用户没说类型，用 AskUserQuestion 问。

### Step 1: 读前——抛起点问题

引导用户根据书名/章标题，写出**起点问题**。

**文学史版：**
```
问题: 读这章之前，你觉得它最核心的问题是什么？
提示: 想想社会背景——当时发生了什么？人们面临什么困境？
选项:
  A) AI 根据标题猜一个（用户可改）
  B) 用户自己写
```

**通用版：**
```
问题: 这本书/这章想回答一个什么问题？
选项:
  A) AI 根据书名猜一个
  B) 用户自己写
```

### Step 2: 读中——配对填写

引导用户读一段，填一张卡片：

**问题卡片**（红色，Color 6）：
```
## ❓ Q{编号}: {问题}
{核心问题一句话}
```

**回答卡片**（绿色，Color 4）：
```
## 💡 {思潮/作者/理论名}
**核心回答**: {一句话}
**代表/关键内容**: {列举}
```

**局限卡片**（橙色，Color 1）：
```
## ⚠️ 局限
{它没回答什么？留下了什么漏洞？}
```

用 AskUserQuestion 逐步引导，每次只生成一张卡片。不要一次性生成所有卡片。

### Step 3: 读后——找洞

引导用户找这个回答的"局限"，这个局限就是下一个起点问题。

### Step 4: 生成 Canvas 文件

每件卡片写成独立的 .md 文件，Canvas 中用 `file` 类型节点引用。这样 Obsidian 会完整渲染 md 内容，不会出现文字截断。

**目录结构：**
```
docs/问题导图/{书名}/
  ├── {书名}.canvas          ← Canvas 文件
  └── cards/
      ├── Q1.md              ← 问题卡片
      ├── A1.md              ← 回答卡片
      ├── L1.md              ← 局限卡片
      ├── Q2.md
      ├── A2.md
      └── ...
```

**卡片文件格式：**

问题卡片（`cards/Q{编号}.md`）：
```markdown
## ❓ Q{编号}: {问题}

{核心问题一句话}

— 由{上一局限}引出
```

回答卡片（`cards/A{编号}.md`）：
```markdown
## 💡 {思潮/作者/理论名}

**核心回答**: {一句话}

**关键内容**:
- {要点1}
- {要点2}
- ...
```

局限卡片（`cards/L{编号}.md`）：
```markdown
## ⚠️ 局限

{它没回答什么？留下了什么漏洞？}
```

**Canvas 布局方案：** 水平流 + 纵向堆叠。每轮 Q→A→L 从左到右一横行展开，多轮从上到下堆叠。每组用 Group 节点包裹，连线带箭头和标签标明流转关系。

**颜色方案：**

| 卡片 | color 值 | Obsidian 色 | 含义 |
|------|----------|-------------|------|
| Q (问题) | Color 6 | 紫色 | 沉思/提问 |
| A (回答) | Color 4 | 绿色 | 回答/生长 |
| L (局限) | Color 2 | 橙色 | 边界/慎思 |

**节点坐标（每轮 `rowIndex` 从 0 开始）：**

每行 y 起始 = `rowIndex * 400 + 60`

| 卡片 | x | y | w | h | type | file 相对路径 |
|------|---|---|---|---|------|-------------|
| Q | 80 | rowY | 420 | 180 | file | `问题导图/{书名}/cards/Q{编号}.md` |
| A | 560 | rowY | 420 | 280 | file | `问题导图/{书名}/cards/A{编号}.md` |
| L | 1040 | rowY | 420 | 180 | file | `问题导图/{书名}/cards/L{编号}.md` |

**Group 节点（包裹每轮）：**

| 字段 | 值 |
|------|-----|
| type | group |
| x | 50 |
| y | rowY - 40 |
| w | 1440 |
| h | 360 |
| label | `"📖 第{N}轮问题链"` |
| color | 偶数轮 Color 5 (青色)，奇数轮 Color 3 (黄色)，交替使用 |

**连线规则（Edge）：**

| 连线 | fromNode | toNode | fromSide | toSide | endArrowhead | label |
|------|----------|--------|----------|--------|--------------|-------|
| Q → A | Q节点id | A节点id | right | left | arrow | "回应" |
| A → L | A节点id | L节点id | right | left | arrow | "局限" |
| L → Q_next | L节点id | 下轮Q节点id | bottom | top | arrow | "引出" |

L→Q_next 的连线建议设置 `path` 参数实现曲线路径，让视觉流转更自然（从 L 底部出发，弧线绕到下一行 Q 顶部）。

**Canvas JSON 结构示例（一轮）：**

```json
{

      "id": "Q1",
      "type": "file",
      "file": "问题导图/{书名}/cards/Q1.md",

      "height": 180,
      "color": "6"
    },
    {
      "id": "A1",
      "type": "file",
      "file": "问题导图/{书名}/cards/A1.md",
      "x": 560,
      "y": 60,
      "width": 420,

    },
    {
      "id": "L1",
      "type": "file",

      "x": 1040,
      "y": 60,
      "width": 420,
      "height": 180,
      "color": "2"
    },
    {
      "id": "group1",
      "type": "group",
      "label": "📖 第1轮问题链",
      "x": 50,
      "y": 20,
      "width": 1440,
      "height": 360,

    }
  ],
  "edges": [
    {
      "id": "e1",
      "fromNode": "Q1",
      "fromSide": "right",
      "toNode": "A1",
      "toSide": "left",
      "endArrowhead": "arrow",

```

**续写逻辑：**
- 读取已有 `.canvas` 文件和 `cards/` 目录
- 找出最大编号，新卡片编号 = 最大编号 + 1
- 新行 y = 最后行 y + 400（注：行 y 指节点 y，不是 group y）
- 新增连线：从上一轮 L 节点底部（fromSide: bottom）→ 新行 Q 节点顶部（toSide: top），endArrowhead: "arrow"，label: "引出"

### Step 5: 完成提示

显示当前问题链摘要 + 建议下一步：

```
✅ 已完成第 {N} 轮问题链

当前链条：
  Q{1}: ... → {回答} → 局限: ...
  Q{2}: ... → {回答} → 局限: ...

建议下一步：
  A) 继续读下一节
  B) 换一本书
  C) 打开 Canvas 文件查看
```

## 重要规则

1. **用户填空，不是 AI 全填。** 回答卡片的"核心回答"和局限由用户自己写，AI 只给骨架和提示选项。
2. **一次只生成一个节点。** 读一段生成一个节点，不要一口气生成整章。
3. **不覆盖已有内容。** 如果 canvas 已存在，只追加节点，不修改已有节点。
4. **允许跳过。** 用户不想填局限？可以跳过，留下空节点后续可补。
5. **颜色编码固定：** Color 6=问题卡片（紫色）, Color 4=回答卡片（绿色）, Color 2=局限卡片（橙色）。
